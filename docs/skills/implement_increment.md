# Skill Doc — Implementar incremento

## Passo a passo

1. Identificar claramente o incremento a implementar
2. Ler PRD.md, FLOW.md, SCHEMA.md e MEMORY.md
3. Delimitar quais arquivos serão alterados
4. Verificar se o schema atual suporta a funcionalidade
5. Implementar a menor solução coerente com os artefatos
6. Executar localmente e testar o fluxo impactado
7. Atualizar MEMORY.md se houver nova decisão

## Regras

- Não expandir escopo por entusiasmo
- Não reescrever áreas não relacionadas ao incremento
- Não introduzir dependência sem necessidade documentada
- Preservar separação entre GUI, services e repositories

## Saída esperada do agente

1. Entendimento do incremento em 2-3 frases
2. Lista dos arquivos que serão alterados
3. Código completo dos módulos modificados
4. Instruções de como executar e testar
5. Limitações restantes e próximo incremento sugerido
